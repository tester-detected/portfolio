# JS-экосистема для QA: Node.js, фреймворки, async

## Node.js

**Node.js** — среда выполнения JavaScript вне браузера. Построена на движке V8 (Chrome).

### Зачем QA знать Node.js

- Cypress и Playwright работают на Node.js
- Конфиги, плагины, хелперы — всё на JS/TS
- npm / yarn — управление зависимостями
- Скрипты автоматизации (парсинг отчётов, интеграция с TMS)

### Node.js vs браузерный JS

| | Браузер | Node.js |
|---|---|---|
| Окружение | Вкладка браузера | Серверный процесс |
| DOM | Есть (`document`, `window`) | Нет |
| Файловая система | Нет доступа | `fs.readFile()` |
| Сеть | `fetch()`, `XMLHttpRequest` | `http`, `https`, `fetch` (v18+) |
| Модули | ES Modules (`import`) | CommonJS (`require`) + ES Modules |

## Фронтенд-фреймворки (обзор для QA)

### React

- Библиотека (не фреймворк) от Meta
- Компонентный подход: UI = дерево компонентов
- Виртуальный DOM — обновляет только изменившиеся части
- JSX — HTML внутри JavaScript
- Однонаправленный поток данных (props сверху вниз)

**Для QA:** элементы могут иметь `data-testid` атрибуты, React DevTools помогают инспектировать компоненты.

### Vue

- Прогрессивный фреймворк (можно подключить частично)
- Реактивная система данных (изменил переменную → UI обновился)
- Шаблоны — ближе к обычному HTML, чем JSX
- Проще порог входа, чем React

### Angular

- Полноценный фреймворк от Google
- TypeScript обязателен
- Dependency Injection, модули, сервисы — всё «из коробки»
- Строгая структура проекта
- Тяжелее для простых проектов

### Сравнение

| | React | Vue | Angular |
|---|---|---|---|
| Тип | Библиотека | Фреймворк | Фреймворк |
| Язык | JS/TS + JSX | JS/TS + Templates | TypeScript |
| Размер | Средний | Маленький | Большой |
| Кривая обучения | Средняя | Низкая | Высокая |
| Экосистема | Огромная | Большая | Большая |

### Node.js vs React — в чём разница?

| | Node.js | React |
|---|---|---|
| Что это | Среда выполнения JS на сервере | Библиотека для UI в браузере |
| Где работает | Сервер | Браузер |
| Для чего | API, скрипты, тулинг | Интерфейс (кнопки, формы, страницы) |
| Связь | React-приложение собирается с помощью Node.js (npm, webpack) |

## Jest — фреймворк для юнит-тестов

**Jest** — тестовый фреймворк для JavaScript/TypeScript от Meta.

### Основные концепции

```javascript
describe('Calculator', () => {           // Группа тестов
  beforeEach(() => {                     // Выполняется перед КАЖДЫМ тестом
    // подготовка
  });

  afterEach(() => {                      // Выполняется после КАЖДОГО теста
    // очистка
  });

  it('should add two numbers', () => {   // Один тест
    expect(add(2, 3)).toBe(5);           // Ассерт
  });

  it('should handle negative', () => {
    expect(add(-1, 1)).toBe(0);
  });
});
```

### Хуки жизненного цикла

| Хук | Когда выполняется |
|-----|------------------|
| `beforeAll` | Один раз перед всеми тестами в `describe` |
| `beforeEach` | Перед каждым тестом |
| `afterEach` | После каждого теста |
| `afterAll` | Один раз после всех тестов |

### Основные матчеры

```javascript
expect(value).toBe(5);              // строгое равенство (===)
expect(value).toEqual({a: 1});      // глубокое сравнение объектов
expect(value).toBeTruthy();         // truthy-значение
expect(value).toBeNull();           // null
expect(value).toContain('hello');   // массив/строка содержит
expect(fn).toThrow();               // функция выбрасывает ошибку
expect(value).toHaveLength(3);      // длина массива/строки
```

## Синхронный и асинхронный код

### Синхронный код

Выполняется последовательно. Каждая строка ждёт завершения предыдущей.

```javascript
console.log('1');
console.log('2');  // выполнится только после '1'
console.log('3');
// Вывод: 1, 2, 3
```

**Аналогия:** очередь в магазине — каждый покупатель ждёт предыдущего.

### Асинхронный код

Операция запускается, но **не блокирует** выполнение. Результат приходит позже.

```javascript
console.log('1');
setTimeout(() => console.log('2'), 1000);  // отложен на 1 сек
console.log('3');
// Вывод: 1, 3, 2
```

**Аналогия:** заказал еду с доставкой — продолжаешь работать, еда приедет когда будет готова.

### Callback → Promise → async/await

```javascript
// 1. Callback (старый способ)
fs.readFile('data.json', (err, data) => {
  if (err) throw err;
  console.log(data);
});

// 2. Promise
fetch('/api/data')
  .then(response => response.json())
  .then(data => console.log(data))
  .catch(error => console.error(error));

// 3. async/await (современный способ)
async function loadData() {
  try {
    const response = await fetch('/api/data');
    const data = await response.json();
    console.log(data);
  } catch (error) {
    console.error(error);
  }
}
```

### Для тестировщика

- Cypress **автоматически** ждёт асинхронных операций (команды встают в очередь)
- Playwright использует `async/await` явно: `await page.click('.btn')`
- В Jest для асинхронных тестов: `async` / `await` или `return promise`
- Понимание async нужно для отладки flaky-тестов (гонки состояний)
