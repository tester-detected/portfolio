# Очереди сообщений, брокеры и асинхронность (Kafka)

## Принципы работы очередей сообщений

Очереди предоставляют буфер для временного хранения сообщений и конечные точки для отправки и получения сообщений в **асинхронном режиме**.

Компонент-**производитель (Producer)** добавляет сообщение в очередь, где оно хранится, пока **потребитель (Consumer)** не извлечёт его и не обработает.

Использование очередей решает две задачи:
- Сокращает время ожидания пользователя (асинхронная обработка)
- Предотвращает потерю информации при сбоях

### Методы получения сообщений

- **Pull** — получатель периодически опрашивает очередь
- **Push** — отправка уведомления получателю при поступлении сообщения (модель «Издатель/Подписчик»)

## Брокер сообщений

**Message Broker** — система, которая занимается сбором и маршрутизацией сообщений. Сообщения передаются с ключом — по нему брокер определяет, в какую очередь направить сообщение.

## Kafka

Kafka — распределённая потоковая платформа для обмена сообщениями. Работает по принципу публикации-подписки (producer-consumer).

### Ключевые понятия

| Понятие | Описание |
|---------|----------|
| **Topic** | Именованный канал для сообщений |
| **Partition** | Раздел внутри топика (для параллелизма) |
| **Producer** | Отправитель сообщений |
| **Consumer** | Получатель сообщений |
| **Offset** | Порядковый номер сообщения в партиции |
| **Broker** | Сервер Kafka, хранящий данные |

### Как тестировать Kafka

1. **Проверка отправки** — убедиться, что producer корректно отправляет сообщение в нужный topic
2. **Проверка получения** — убедиться, что consumer читает сообщение и обрабатывает его правильно
3. **CLI-утилиты:** `kafka-console-producer`, `kafka-console-consumer` для ручной отправки/просмотра
4. **Некорректные сообщения** — тестировать обработку невалидных данных и «отравленных писем» (poison pills)

## Синхронные vs Асинхронные запросы

### Синхронные (блокирующие)

Программа отправляет запрос и **ждёт**, пока сервер не обработает его и не вернёт ответ.

**Аналогия:** звонок по телефону — вы ждёте ответа собеседника.

**Примеры:** загрузка страницы (браузер ждёт HTML, CSS и JS).

### Асинхронные (неблокирующие)

Программа отправляет запрос, получает подтверждение о принятии, **не дожидаясь** результата, и продолжает работу.

**Аналогия:** отправка письма по электронной почте — отправили и работаете дальше.

**Примеры в коде:**
- AJAX/Fetch: `fetch('/api/data').then(response => response.json());`
- Node.js: чтение файла с `fs.readFile`
- Веб-хуки (Webhook), очереди сообщений (RabbitMQ, Kafka)
- Long Polling, события в браузерах
