# Техники тест-дизайна — Форма авторизации

Практическое применение техник тест-дизайна на реальной фиче: **форма входа** с полями email и пароль.

## Спецификация

| Элемент | Правила |
|---------|---------|
| **Email** | Обязательное. Формат `user@domain.tld`. Максимум 254 символа. Регистронезависимый. |
| **Пароль** | Обязательное. 8–64 символа. Минимум одна заглавная буква, одна строчная, одна цифра. |
| **Кнопка "Войти"** | Активна только когда оба поля заполнены. |
| **"Запомнить меня"** | Чекбокс. Если включён — сессия 30 дней, иначе — до закрытия браузера. |
| **Блокировка** | После 5 неудачных попыток подряд аккаунт блокируется на 15 минут. |
| **Сообщения об ошибках** | Неверные данные: «Неверный email или пароль» (без подсказки, какое поле неправильное). Заблокирован: «Аккаунт временно заблокирован. Попробуйте через 15 минут.» |

---

## 1. Классы эквивалентности (Equivalence Partitioning)

Разбиваем входные данные на группы (классы), внутри которых система ведёт себя одинаково. Тестируем по одному значению из каждого класса вместо перебора всех возможных вариантов.

### Email

| # | Класс | Пример | Ожидание |
|---|-------|--------|----------|
| 1 | Валидный email | `user@example.com` | Принят |
| 2 | Без `@` | `userexample.com` | Ошибка валидации |
| 3 | Без домена | `user@` | Ошибка валидации |
| 4 | Без локальной части | `@example.com` | Ошибка валидации |
| 5 | Пустая строка | *(пусто)* | Обязательное поле |
| 6 | Только пробелы | `"   "` | Обязательное поле |
| 7 | Валидный, но незарегистрированный | `unknown@example.com` | «Неверный email или пароль» |

### Пароль

| # | Класс | Пример | Ожидание |
|---|-------|--------|----------|
| 1 | Валидный (все правила соблюдены) | `Passw0rd` | Принят |
| 2 | Без заглавной буквы | `passw0rd` | Ошибка валидации |
| 3 | Без строчной буквы | `PASSW0RD` | Ошибка валидации |
| 4 | Без цифры | `Password` | Ошибка валидации |
| 5 | Слишком короткий (< 8) | `Pa1` | Ошибка валидации |
| 6 | Слишком длинный (> 64) | строка из 65 символов | Ошибка валидации |
| 7 | Пустая строка | *(пусто)* | Обязательное поле |

## 2. Граничные значения (Boundary Value Analysis)

Тестируем на точных границах допустимых диапазонов — именно там живёт большинство багов.

### Длина пароля

| # | Длина | Значение | Ожидание |
|---|-------|----------|----------|
| 1 | 0 | *(пусто)* | Обязательное поле |
| 2 | 1 | `A` | Слишком короткий |
| 3 | 7 | `Abcde1x` | Слишком короткий |
| 4 | **8** | `Abcdef1x` | **Валиден (нижняя граница)** |
| 5 | 9 | `Abcdef1xx` | Валиден |
| 6 | 63 | строка из 63 символов | Валиден |
| 7 | **64** | строка из 64 символов | **Валиден (верхняя граница)** |
| 8 | 65 | строка из 65 символов | Слишком длинный |

### Длина email

| # | Длина | Ожидание |
|---|-------|----------|
| 1 | 1 символ (например, `a`) | Невалидный формат |
| 2 | 6 символов — минимальный валидный: `a@b.co` | Валиден |
| 3 | 254 символа | **Валиден (верхняя граница)** |
| 4 | 255 символов | Слишком длинный |

### Неудачные попытки входа (блокировка)

| # | Попытка | Ожидание |
|---|---------|----------|
| 1 | 1-я неудачная | «Неверный email или пароль» |
| 2 | 4-я неудачная | «Неверный email или пароль» |
| 3 | **5-я неудачная** | **Аккаунт заблокирован** |
| 4 | 6-я попытка (верный пароль) | Всё ещё заблокирован |
| 5 | Попытка через 14 мин 59 сек | Всё ещё заблокирован |
| 6 | **Попытка через 15 мин 00 сек** | **Разблокирован, вход успешен** |

## 3. Таблица решений (Decision Table)

Сопоставляем комбинации условий с ожидаемыми результатами. Полезно, когда несколько входных данных влияют друг на друга.

### Результаты попытки входа

| # | Email валиден? | Email зарегистрирован? | Пароль верный? | Попыток < 5? | Запомнить? | Результат |
|---|:---:|:---:|:---:|:---:|:---:|-----------|
| 1 | Нет | — | — | — | — | Ошибка валидации |
| 2 | Да | Нет | — | — | — | «Неверный email или пароль» |
| 3 | Да | Да | Нет | Да | — | «Неверный email или пароль», счётчик +1 |
| 4 | Да | Да | Нет | Нет | — | «Аккаунт временно заблокирован» |
| 5 | Да | Да | Да | Да | Выкл | Вход OK, сессия до закрытия браузера |
| 6 | Да | Да | Да | Да | Вкл | Вход OK, сессия 30 дней |
| 7 | Да | Да | Да | Нет | — | «Аккаунт временно заблокирован» |

> **Важный инсайт (строка 7):** Даже правильный пароль отклоняется, пока аккаунт заблокирован — это нужно тестировать явно.

## 4. Диаграмма переходов состояний (State Transition)

Моделирует, как система переходит между состояниями в зависимости от событий.

```
                    ┌──────────────────────────────────────┐
                    │                                      │
                    v                                      │
              ┌───────────┐                                │
              │   ФОРМА    │                                │
              │   ВХОДА    │                                │
              └─────┬──────┘                                │
                    │                                      │
                    │ Отправка                              │
                    v                                      │
              ┌───────────┐                                │
              │ ВАЛИДАЦИЯ  │                                │
              └──┬────┬───┘                                │
                 │    │                                    │
       Невалидно │    │ Валидно                            │
                 │    v                                    │
                 │  ┌──────────────┐                       │
                 │  │АУТЕНТИФИКАЦИЯ│                       │
                 │  └──┬───┬───┬──┘                       │
                 │     │   │   │                           │
                 │     │   │   │ Данные верны              │
                 │     │   │   └──────────────────┐        │
                 │     │   │                      v        │
                 │     │   │               ┌──────────┐    │
                 │     │   │               │  ВХОД OK  │    │
                 │     │   │               └──────────┘    │
                 │     │   │                               │
                 │     │   │ Неверный пароль               │
                 │     │   │ (попытка < 5)                 │
                 │     │   v                               │
                 │     │ ┌───────────────┐                 │
                 │     │ │ ОШИБКА        │── Повтор ───────┘
                 │     │ │ (попыток: N)  │
                 │     │ └───────────────┘
                 │     │
                 │     │ 5-я неудачная попытка
                 │     v
                 │   ┌────────────┐   Через 15 мин    ┌───────────┐
                 │   │ БЛОКИРОВКА │──────────────────> │  ФОРМА    │
                 │   └────────────┘                    │  ВХОДА    │
                 │                                     │ (счётчик  │
                 v                                     │  сброшен) │
          ┌──────────────┐                             └───────────┘
          │   ОШИБКА     │── Исправить ────────────────────┘
          │  ВАЛИДАЦИИ   │
          └──────────────┘
```

### Таблица переходов

| Текущее состояние | Событие | Следующее состояние | Действие |
|-------------------|---------|---------------------|----------|
| Форма входа | Отправка | Валидация | Проверка формата полей |
| Валидация | Невалидные данные | Ошибка валидации | Показать ошибки полей |
| Валидация | Валидные данные | Аутентификация | Запрос на сервер |
| Ошибка валидации | Исправление + повтор | Валидация | Перепроверка |
| Аутентификация | Данные верны | Вход OK | Создать сессию, редирект |
| Аутентификация | Неверный пароль (N < 5) | Ошибка | Показать ошибку, счётчик +1 |
| Аутентификация | Неверный пароль (N = 5) | Блокировка | Показать сообщение о блокировке |
| Ошибка | Повтор | Валидация | Перепроверка |
| Блокировка | Прошло 15 мин | Форма входа | Сброс счётчика |
| Блокировка | Отправка (любая) | Блокировка | Показать сообщение о блокировке |

## 5. Попарное тестирование (Pairwise)

При тестировании комбинаций нескольких параметров полный перебор растёт экспоненциально. Pairwise гарантирует, что каждая пара значений параметров встречается хотя бы в одном тесте — ловит большинство багов взаимодействия при значительно меньшем числе тестов.

### Параметры

| Параметр | Значения |
|----------|----------|
| Браузер | Chrome, Firefox, Safari |
| ОС | Windows, macOS, Linux |
| Запомнить меня | Вкл, Выкл |
| Данные для входа | Верные, Неверные |

Полный перебор: 3 x 3 x 2 x 2 = **36 тестов**  
Pairwise: **9 тестов** (сокращение на 75%)

### Набор тестов

| # | Браузер | ОС | Запомнить | Данные | Ожидание |
|---|---------|------|-----------|--------|----------|
| 1 | Chrome | Windows | Вкл | Верные | Вход OK, сессия 30 дней |
| 2 | Chrome | macOS | Выкл | Неверные | Сообщение об ошибке |
| 3 | Chrome | Linux | Вкл | Неверные | Сообщение об ошибке |
| 4 | Firefox | Windows | Выкл | Неверные | Сообщение об ошибке |
| 5 | Firefox | macOS | Вкл | Верные | Вход OK, сессия 30 дней |
| 6 | Firefox | Linux | Выкл | Верные | Вход OK, сессия до закрытия |
| 7 | Safari | Windows | Вкл | Неверные | Сообщение об ошибке |
| 8 | Safari | macOS | Выкл | Верные | Вход OK, сессия до закрытия |
| 9 | Safari | Linux | Вкл | Верные | Вход OK, сессия 30 дней |

> Каждая пара значений (например, Chrome + macOS, Firefox + Выкл, Safari + Верные) встречается минимум в одном тесте.

## Чек-лист

Легковесная альтернатива детальным тест-кейсам — для регресса и исследовательского тестирования.

### Функциональность

- [ ] Вход с валидными данными
- [ ] Вход с незарегистрированным email
- [ ] Вход с неверным паролем
- [ ] Вход с пустым email
- [ ] Вход с пустым паролем
- [ ] Вход с обоими пустыми полями
- [ ] Кнопка «Войти» неактивна при пустых полях
- [ ] «Запомнить меня» включён — сессия сохраняется после перезапуска браузера
- [ ] «Запомнить меня» выключен — сессия завершается при закрытии браузера
- [ ] Блокировка аккаунта после 5 неудачных попыток
- [ ] Отображение сообщения о блокировке
- [ ] Разблокировка через 15 минут
- [ ] Успешный вход сбрасывает счётчик неудачных попыток
- [ ] Сообщение об ошибке не раскрывает, какое именно поле неверное

### Валидация

- [ ] Email без `@`
- [ ] Email без домена
- [ ] Email с пробелами
- [ ] Email > 254 символов
- [ ] Email регистронезависимый (`User@Example.COM` = `user@example.com`)
- [ ] Пароль 7 символов (ниже минимума)
- [ ] Пароль 8 символов (нижняя граница)
- [ ] Пароль 64 символа (верхняя граница)
- [ ] Пароль 65 символов (выше максимума)
- [ ] Пароль без заглавной буквы
- [ ] Пароль без строчной буквы
- [ ] Пароль без цифры
- [ ] SQL-инъекция в поле email (`' OR 1=1 --`)
- [ ] XSS в поле email (`<script>alert(1)</script>`)

### UI / UX

- [ ] Пароль замаскирован по умолчанию
- [ ] Кнопка «Показать пароль» работает
- [ ] Enter отправляет форму
- [ ] Tab-порядок: Email → Пароль → Запомнить меня → Войти
- [ ] Сообщения об ошибках видимы и читаемы
- [ ] Состояние загрузки на кнопке «Войти» во время запроса
- [ ] Форма корректно отображается на мобильных (адаптивность)
