# Docker, Kubernetes, контейнеры и виртуальные машины

## Контейнер vs Виртуальная машина

```
Виртуальная машина:                  Контейнер:

┌──────────┐ ┌──────────┐           ┌──────┐ ┌──────┐ ┌──────┐
│  App A   │ │  App B   │           │App A │ │App B │ │App C │
├──────────┤ ├──────────┤           ├──────┤ ├──────┤ ├──────┤
│ Guest OS │ │ Guest OS │           │ Libs │ │ Libs │ │ Libs │
├──────────┴─┴──────────┤           ├──────┴─┴──────┴─┴──────┤
│      Hypervisor       │           │     Container Engine    │
├───────────────────────┤           │        (Docker)         │
│       Host OS         │           ├────────────────────────┤
├───────────────────────┤           │        Host OS          │
│      Hardware         │           ├────────────────────────┤
└───────────────────────┘           │       Hardware          │
                                    └────────────────────────┘
```

| | Контейнер | Виртуальная машина |
|---|---|---|
| Изоляция | На уровне процесса (общее ядро ОС) | Полная (свой ядро + ОС) |
| Размер | Мегабайты (10-200 МБ) | Гигабайты (1-20 ГБ) |
| Запуск | Секунды | Минуты |
| Потребление ресурсов | Минимальное | Значительное |
| Портативность | Образ = файл, запускается везде | Тяжелее переносить |
| Безопасность | Слабее (общее ядро) | Сильнее (полная изоляция) |
| Примеры | Docker, Podman | VMware, VirtualBox, Hyper-V |

## Docker

**Docker** — платформа для создания, развёртывания и запуска приложений в контейнерах.

### Основные понятия

| Понятие | Описание |
|---------|----------|
| **Image (Образ)** | Шаблон для создания контейнера. Содержит ОС, зависимости, код. Неизменяемый. |
| **Container** | Запущенный экземпляр образа. Изолированный процесс. |
| **Dockerfile** | Инструкция для сборки образа (FROM, RUN, COPY, CMD) |
| **Docker Compose** | Инструмент для запуска нескольких контейнеров (docker-compose.yml) |
| **Docker Hub** | Реестр (registry) образов — как GitHub для кода |
| **Volume** | Постоянное хранилище данных (данные не теряются при перезапуске контейнера) |

### Основные команды

```bash
# Образы
docker build -t myapp .            # Собрать образ из Dockerfile
docker images                       # Список образов
docker pull nginx                   # Скачать образ из Docker Hub

# Контейнеры
docker run -d -p 8080:80 nginx      # Запустить в фоне, проброс порта
docker ps                           # Список запущенных контейнеров
docker ps -a                        # Все контейнеры (включая остановленные)
docker stop <id>                    # Остановить
docker rm <id>                      # Удалить
docker logs <id>                    # Посмотреть логи
docker exec -it <id> bash           # Зайти внутрь контейнера

# Docker Compose
docker compose up -d                # Запустить все сервисы
docker compose down                 # Остановить и удалить
docker compose logs -f              # Логи в реальном времени
```

## Kubernetes (K8s)

**Kubernetes** — система оркестрации контейнеров. Управляет развёртыванием, масштабированием и отказоустойчивостью контейнеризированных приложений.

### Docker vs Kubernetes

| | Docker | Kubernetes |
|---|---|---|
| Что делает | Создаёт и запускает контейнеры | Управляет кластером контейнеров |
| Масштаб | 1 сервер | Десятки-сотни серверов |
| Масштабирование | Вручную | Автоматическое (HPA) |
| Самовосстановление | Нет | Да (перезапуск упавших подов) |
| Балансировка | Нет (нужен nginx вручную) | Встроенная (Service) |
| Конфигурация | docker-compose.yml | YAML-манифесты (Deployment, Service) |
| Аналогия | Один корабль | Флот кораблей с адмиралом |

### Основные понятия K8s

| Понятие | Описание |
|---------|----------|
| **Pod** | Минимальная единица — один или несколько контейнеров |
| **Deployment** | Описание: какие поды, сколько реплик, как обновлять |
| **Service** | Стабильный IP/DNS для доступа к подам (балансировщик) |
| **Node** | Физический или виртуальный сервер в кластере |
| **Cluster** | Набор нод, управляемых K8s |
| **Namespace** | Логическое разделение ресурсов (dev, staging, prod) |
| **Ingress** | Маршрутизация HTTP-трафика извне в кластер |

### Для тестировщика

- Docker: умеем запускать тесты в контейнерах (Cypress в Docker — из твоего проекта)
- K8s: понимаем, как приложение развёрнуто на тестовом стенде
- `kubectl get pods` — проверить, что поды живы
- `kubectl logs <pod>` — посмотреть логи приложения
- `kubectl describe pod <pod>` — диагностика проблем
