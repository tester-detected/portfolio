# Виды требований к ПО

## Типы требований

- **Функциональные** — описывают, что именно ПО должно делать (функции, действия, обработка данных). Пример: «система должна позволять пользователю регистрироваться».
- **Нефункциональные** — определяют характеристики качества и ограничения: производительность, надёжность, безопасность, удобство использования, совместимость.
- **Бизнес-требования** — описывают, зачем организации нужна эта система (цели, выгода).
- **Пользовательские требования** — описывают задачи, которые пользователи должны выполнять с помощью системы.

## Ключевые характеристики качественных требований

| Характеристика | Описание |
|----------------|----------|
| **Однозначность** | Требование не допускает разных трактовок |
| **Выполнимость** | Требования возможно реализовать |
| **Проверяемость / Тестируемость** | Можно протестировать выполнение |
| **Полнота** | Содержит всю необходимую информацию |
| **Прослеживаемость** | Можно отследить от бизнес-цели до реализации в коде |
| **Актуальность** | Требование соответствует текущим потребностям |
| **Атомарность** | Описывает только одну ситуацию |

## Примеры

- **Функциональное:** «При нажатии кнопки "Оплатить" система отправляет запрос в банк»
- **Нефункциональное (производительность):** «Время отклика системы не превышает 2 секунд»
- **Нефункциональное (безопасность):** «Все пароли хранятся в зашифрованном виде»
