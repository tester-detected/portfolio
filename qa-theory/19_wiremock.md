# WireMock — заглушки для тестирования

## Что такое WireMock

**WireMock** — инструмент для создания заглушек (mock/stub) HTTP-сервисов. Имитирует API, которое ещё не готово, недоступно или нестабильно.

## Зачем нужны заглушки

- Бэкенд ещё не написан, а фронт уже нужно тестировать
- Внешний сервис (платёжная система, SMS-шлюз) недоступен на тесте
- Нужно воспроизвести редкие ситуации (таймаут, ошибка 500, пустой ответ)
- Интеграционные тесты не должны зависеть от внешних сервисов
- Нужна стабильность — реальный API может менять данные

## Как работает

```
Без WireMock:
  Приложение → Реальный API (может быть недоступен)

С WireMock:
  Приложение → WireMock (localhost:8080) → возвращает заранее настроенные ответы
```

## Основные понятия

| Понятие | Описание |
|---------|----------|
| **Stub (Заглушка)** | Правило: «если пришёл такой запрос — верни такой ответ» |
| **Request matching** | Как WireMock определяет, к какому stub относится запрос (URL, метод, заголовки, тело) |
| **Response definition** | Что вернуть: статус, заголовки, тело |
| **Verification** | Проверка: был ли запрос отправлен (и сколько раз) |

## Пример конфигурации (JSON)

```json
{
  "request": {
    "method": "GET",
    "urlPathPattern": "/api/products/[0-9]+"
  },
  "response": {
    "status": 200,
    "headers": {
      "Content-Type": "application/json"
    },
    "jsonBody": {
      "id": 42,
      "name": "Тестовый товар",
      "price": 1500,
      "stock": 10
    }
  }
}
```

## Типичные сценарии

### Имитация ошибки сервера
```json
{
  "request": { "method": "POST", "url": "/api/orders" },
  "response": {
    "status": 500,
    "jsonBody": { "error": "Internal Server Error" }
  }
}
```

### Имитация таймаута
```json
{
  "request": { "method": "GET", "url": "/api/products" },
  "response": {
    "status": 200,
    "fixedDelayMilliseconds": 30000,
    "jsonBody": { "items": [] }
  }
}
```

### Имитация пустого ответа
```json
{
  "request": { "method": "GET", "url": "/api/products" },
  "response": {
    "status": 200,
    "jsonBody": { "items": [], "total": 0 }
  }
}
```

## Альтернативы WireMock

| Инструмент | Язык/платформа | Особенности |
|-----------|---------------|-------------|
| **WireMock** | Java, standalone | Мощный, много функций |
| **MockServer** | Java | Похож на WireMock |
| **json-server** | Node.js | Быстрый REST API из JSON-файла |
| **Prism** | Node.js | Mock из OpenAPI/Swagger спецификации |
| **Charles Map Local** | Десктоп | Подмена ответов через прокси |
| **MSW (Mock Service Worker)** | JS/TS (браузер) | Перехват fetch/XHR в браузере |

## Для тестировщика

- Уметь настроить заглушку для изоляции тестируемого компонента
- Понимать разницу: **mock** (проверяем, был ли вызов) vs **stub** (возвращаем фиксированный ответ)
- Использовать заглушки для воспроизведения edge-cases (ошибки, пустые данные, таймауты)
