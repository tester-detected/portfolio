# Web Vitals и Lighthouse

## Core Web Vitals

**Core Web Vitals** — набор метрик от Google, которые оценивают пользовательский опыт на странице. Влияют на SEO-ранжирование.

### Три основные метрики

| Метрика | Что измеряет | Хорошо | Нужно улучшить | Плохо |
|---------|-------------|:---:|:---:|:---:|
| **LCP** (Largest Contentful Paint) | Скорость загрузки главного контента | < 2.5 сек | 2.5 — 4 сек | > 4 сек |
| **INP** (Interaction to Next Paint) | Отзывчивость на действия пользователя | < 200 мс | 200 — 500 мс | > 500 мс |
| **CLS** (Cumulative Layout Shift) | Визуальная стабильность (прыгает ли контент) | < 0.1 | 0.1 — 0.25 | > 0.25 |

### LCP — Largest Contentful Paint

Время до отображения самого большого видимого элемента (картинка, заголовок, видео).

**Что влияет на LCP:**
- Медленный серверный ответ (TTFB)
- Тяжёлые CSS/JS, блокирующие рендеринг
- Большие изображения без оптимизации
- Отложенная загрузка основного контента

### INP — Interaction to Next Paint

Задержка между действием пользователя (клик, нажатие клавиши) и визуальным обновлением страницы. Заменил FID (First Input Delay) в 2024 году.

**Что влияет на INP:**
- Тяжёлые обработчики событий
- Большой JavaScript, блокирующий main thread
- Сложные перерисовки DOM

### CLS — Cumulative Layout Shift

Насколько контент «прыгает» при загрузке. Например, баннер загрузился и сдвинул текст вниз.

**Типичные причины плохого CLS:**
- Картинки без указания `width` и `height`
- Динамически вставляемая реклама
- Шрифты, загружающиеся позже (FOUT)
- Контент, появляющийся без зарезервированного места

---

## Другие метрики производительности

| Метрика | Что измеряет |
|---------|-------------|
| **FCP** (First Contentful Paint) | Время до первого отображённого контента |
| **TTFB** (Time to First Byte) | Время до первого байта от сервера |
| **TTI** (Time to Interactive) | Когда страница становится полностью интерактивной |
| **TBT** (Total Blocking Time) | Общее время, когда main thread заблокирован |
| **Speed Index** | Как быстро видимая область заполняется контентом |

## Lighthouse

**Lighthouse** — встроенный инструмент Chrome для аудита веб-страниц по 5 категориям.

### Как запустить

1. **DevTools** → вкладка Lighthouse → выбрать категории → Generate report
2. **Командная строка**: `npx lighthouse https://example.com --view`
3. **PageSpeed Insights**: https://pagespeed.web.dev

### 5 категорий аудита

| Категория | Что проверяет | Пример рекомендаций |
|-----------|-------------|---------------------|
| **Performance** | Скорость загрузки и интерактивность | Сжать изображения, убрать неиспользуемый CSS |
| **Accessibility** | Доступность для людей с ограничениями | Alt-текст у картинок, контраст цветов, ARIA-атрибуты |
| **Best Practices** | Современные стандарты веб-разработки | HTTPS, нет ошибок в консоли, нет deprecated API |
| **SEO** | Поисковая оптимизация | Meta-теги, robots.txt, адаптивная вёрстка |
| **PWA** | Progressive Web App | Service Worker, manifest.json, оффлайн-режим |

### Результат Lighthouse

Каждая категория получает оценку от 0 до 100:
- 90-100 — Зелёный (хорошо)
- 50-89 — Оранжевый (нужно улучшить)
- 0-49 — Красный (плохо)

### Что тестировщику проверять

- Performance > 70 на мобильных устройствах
- LCP < 2.5 сек на основных страницах
- CLS < 0.1 — контент не прыгает
- Accessibility > 90 — базовая доступность
- Нет ошибок в консоли (Best Practices)
- Результаты стабильны (запускать 3-5 раз, брать медиану)

### Важно: Lighthouse ≠ реальные пользователи

Lighthouse — это **лабораторные** данные (синтетика). Реальные метрики (field data) собираются через:
- Chrome User Experience Report (CrUX)
- Google Search Console → Core Web Vitals
- RUM-инструменты (Real User Monitoring)
