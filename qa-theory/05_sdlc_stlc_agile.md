# Жизненный цикл ПО (SDLC), STLC, Agile

## SDLC — Жизненный цикл ПО

SDLC — это алгоритм создания IT-продукта, с момента принятия решения о его разработке до момента, когда ПО перестают использовать.

Каждый этап опирается на результат предыдущего и даёт необходимые указания для выполнения последующего. Основная функция SDLC — регламентирование и формализация процесса разработки.

### Основные этапы SDLC

1. **Планирование и анализ требований** — определение целей, ограничений, ресурсов и анализ потребностей пользователей
2. **Проектирование архитектуры** — разработка структуры системы, баз данных, интерфейсов
3. **Разработка (кодирование)** — написание программного кода на основе технического задания
4. **Тестирование** — поиск ошибок и проверка соответствия требованиям
5. **Внедрение (развёртывание)** — выпуск продукта, установка, настройка для пользователей
6. **Сопровождение и поддержка** — устранение ошибок, обновление, добавление новых функций

### Популярные модели жизненного цикла

- **Каскадная (Waterfall)** — последовательное выполнение этапов, переход к следующему только после завершения предыдущего
- **Итеративная (Iterative)** — разработка циклами, продукт совершенствуется от версии к версии
- **Спиральная (Spiral)** — упор на анализ рисков на каждом витке разработки
- **Agile (Гибкая)** — быстрая итеративная разработка с частой поставкой функционала и быстрой адаптацией к изменениям

## Agile

### Ценности Agile-манифеста

- **Люди важнее** процессов и стандартных инструментов
- **Создание продукта важнее**, чем подбор и согласование документации
- **Услышать позицию клиента** и сотрудничать с ним превыше контракта
- **Вносить коррективы важнее**, чем изначально разработанная стратегия

### Agile подходы: Scrum и Kanban

| | Scrum | Kanban |
|---|---|---|
| **Итерации** | Фиксированные спринты (1-2 недели) | Непрерывный поток |
| **Планирование** | В начале спринта | Задачи добавляются в любой момент |
| **Изменения** | Нельзя менять задачи в текущем спринте | Можно добавлять задачи когда угодно |
| **Роли** | Product Owner, Scrum Master, Team | Нет специальных ролей |
| **Ограничения** | Объём работ в спринте | WIP-лимиты (количество незавершённых задач) |
| **Метрика успеха** | Velocity (скорость команды за спринт) | Cycle Time (время выполнения задачи) |

## STLC — Жизненный цикл тестирования

### Основные этапы STLC

1. **Анализ требований** (Requirements Analysis) — изучение требований для определения области тестирования, входных и выходных критериев
2. **Планирование тестирования** (Test Planning) — определение стратегии, ресурсов, сроков и создание тест-плана
3. **Дизайн тестов** (Test Design) — разработка тест-кейсов, тестовых сценариев и других артефактов
4. **Подготовка окружения** (Test Environment Setup) — настройка тестовой среды
5. **Выполнение тестов** (Test Execution) — запуск тест-кейсов и фиксация результатов
6. **Закрытие цикла** (Test Cycle Closure) — анализ результатов, отчёты, сбор уроков
